~Доступные ключи коммандной строки:
/e:=%FileName% -> Выставляет значение полю "Etalone file"
/p:=%FileName% -> Выставляет значение полю "Patched file"
/o:=%FileName% -> Выставляет значение полю "Output file"
/t:=%Text%     -> Выставляет значение полю "Project's title"
/i:=%FileName% -> Выставляет значение полю "Project's icon"
/*:=%Pattern%  -> Выставляет значение полю "Patching target"
/r:=%RegPath%  -> Выставляет значение полю "Registry patholder"
/a:=%Text%     -> Выставляет значение полю "Additional info"
/now -> Отправляет полученные данные на обработку. Триггер.
/cli -> Отправляет полученные данные на обработку, после чего программа прекращает работу. Триггер.
/cli:silent -> Отправляет полученные данные на "бесшумную" обработку, после чего программа прекращает работу. Триггер.

Примечание #1: не попадающие в вышеуказанный список аргументы будут использованы для заполнения свободных полей.
Примечание #2: для корректной отправки содержащего пробелы аргументов, следует заключить их в кавычки или апострофы.
Примечание #3: для корректной отправки содержащих апострофы и кавычки текста, следует предварять их символом `
Примечание #4: для корректной отправки знаков переноса строки, следует заменять их на комбинацию `n
Примечание #5: результаты "бесшумной" генерации патчера пишутся в стандартный выходной поток.
Примечание #6: командная строка не должна содержать более одного аргумента-триггера.
